### 1. CHROMA_STR from HWY_ACHROM_FRAC (x=202)

**Verdict: REJECT**

#### Hypothesis recap
Scale chroma lift down in gray-heavy scenes:
```hlsl
chroma_str *= lerp(1.0, 0.70, achrom_frac);
```

#### Signal trace

`HWY_ACHROM_FRAC` (slot 202) is written by `analysis_frame.fx:315–316`:
```hlsl
if (xi == HWY_ACHROM_FRAC)
    return float4(tex2Dlod(MeanChromaSamp, float4(0.5, 0.5, 0, 0)).a, 0.0, 0.0, 1.0);
```

The `.a` channel of `MeanChromaTex` is `achrom_frac`, computed in `MeanChromaPS`
(`analysis_frame.fx:421–463`):
```hlsl
achrom_count += 1.0 - in_b;   // in_b = step(0.05, C)
// ...
float achrom_frac = achrom_count / float(DS_W * DS_H);
```

Definition: **fraction of 32×18 downsampled pixels with Oklab C < 0.05.**

EMA smoothing (`analysis_frame.fx:466`):
```hlsl
float alpha = saturate(frametime * 0.005);
```
At 60 fps (frametime ≈ 16.7 ms): α ≈ 0.083, giving τ ≈ 12 frames (~0.5 s).

#### Current chroma_str computation

`grade.fx:570–572`:
```hlsl
float chroma_str = CHROMA_STR * 0.04;
chroma_str *= lerp(1.0, 0.65, smoothstep(0.02, 0.08, local_var));   // R68A — texture regions
chroma_str *= lerp(0.80, 1.20, smoothstep(0.05, 0.35, zone_log_key)); // R117 — Hunt luminance
```

The Hunt term already scales from **0.80× (dark scenes)** to **1.20× (bright scenes)**
based on `zone_log_key`.

#### Structural confound: dark pixels ≠ neutral pixels

The Oklab C threshold (0.05) captures two distinct populations:

| Scene type | achrom_frac behavior | zone_log_key | Hunt factor |
|---|---|---|---|
| Overcast / gray interior | High — genuine neutrality | Medium (0.15–0.25) | ~1.0× |
| Dark corridor / shadow-heavy | High — *Oklab C → 0 in shadows regardless of surface color* | Low (0.05–0.12) | 0.80× |
| Bright colorful exterior | Low | High (0.25–0.40) | 1.10–1.20× |

In a dark scene Oklab C naturally approaches zero across all surfaces — a red brick wall
at linear luminance 0.03 has C ≈ 0.01–0.02, well below the 0.05 threshold. This inflates
`achrom_frac` in precisely the scenes where Hunt already fires (low `zone_log_key` → 0.80×).

Applying the proposed formula in a dark scene:
- Hunt: 0.80×
- achrom_frac (elevated because dark): lerp(1.0, 0.70, high_value) ≈ 0.75×
- Combined: **0.60×** of CHROMA_STR

The chroma lift is most physiologically meaningful in dark scenes (Hunt effect, low-C
shadows). Doubly suppressing it by both scalings undermines the core purpose of
`LiftChroma`.

#### The overcast case is benign without the fix

`LiftChroma` (`grade.fx:231–234`) already self-limits in neutral scenes:
```hlsl
float t = saturate(1.0 - C / max(pivot, 0.001));
return C * (1.0 + strength * t * t);
```
When C is near zero (achromatic pixels), `t ≈ 1` and lift fires — but this lifts truly
gray pixels toward the band pivot (~0.08–0.15). There is no "over-saturation" in a gray
scene; achromatic pixels lifted to C ≈ 0.02–0.04 remain perceptually neutral. The
concern in the job file is not confirmed by the LiftChroma math.

#### Temporal lag risk

The 0.5 s EMA time constant creates a one-sided transient on cuts: cutting from a gray
interior to a colorful exterior, `achrom_frac` decays slowly, suppressing chroma lift for
~12 frames into the new shot. The scene-cut detection at `HWY_SCENE_CUT` (slot 199) is
not wired to flush this EMA, so the transient is structural.

#### Gate-free fix unavailable

Separating genuine neutrality from luminance-induced desaturation requires a
luminance-normalized achrom measure (e.g., fraction of mid-luminance pixels with C < 0.05).
This is not on the highway and computing it would add a per-pixel pass. Any attempt to
salvage the signal with a zone_log_key gate violates the "no gates" rule.

#### Conclusion

`HWY_ACHROM_FRAC` conflates luminance-induced desaturation with genuine scene neutrality.
The proposed scaling double-counts with the existing Hunt luminance term and would
suppress chroma lift in dark scenes — the opposite of the intent. The overcast
over-saturation concern is not supported by the LiftChroma math. **Reject.**
