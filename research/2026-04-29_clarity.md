# Research Findings — Local Contrast & Clarity — 2026-04-29

## Search angle
Investigated spatially-selective contrast enhancement from computational photography, perceptual vision science (CSF theory), and real-time GPU graphics. Cross-field angle: whether the human visual system's characteristic bandpass spatial sensitivity profile — peaking at 3–5 cycles per degree — can be operationalized as a self-limiting, gate-free clarity operation that reuses the existing bilateral blur infrastructure already computed by `pro_mist.fx`.

## Finding 1: Bilateral Residual Clarity — Repurposing pro_mist's Laplacian Layer

**Source:** https://arxiv.org/abs/2502.13945 — "GPU-Friendly Laplacian Texture Blending"
**Year:** 2025 (February)
**Field:** Computational photography / real-time GPU graphics

### Core thesis
Local Laplacian filters decompose images into pyramid coefficients by remapping each level independently; the heart of the operation is `Laplacian[l] = image[l] − Gaussian_blur[l]`. The Feb 2025 paper shows this runs real-time using standard GPU texture infrastructure with no precomputation. Applied to our pipeline: `pro_mist.fx`'s bilateral separable blur already produces a Gaussian-family low-pass image in `DiffuseTex`. The residual `BackBuffer − DiffuseTex` is a Laplacian coefficient that the shader computes and then silently discards inside the final `lerp`. Positively blending this residual back — rather than subtracting it — converts pro_mist's softening math into a clarity effect, with zero additional GPU passes.

### Current code baseline
File: `general/pro-mist/pro_mist.fx`, DiffuseV pass (~lines 183–197)
```hlsl
    float3 diffused = tex2D(DiffuseSamp, uv).rgb * GW[0];
    float  w        = GW[0];

    [loop]
    for (int i = 1; i <= 4; i++)
    {
        float3 tp = tex2D(DiffuseSamp, uv + ds * i).rgb;
        float3 tn = tex2D(DiffuseSamp, uv - ds * i).rgb;
        float  wp = BilateralW(lc, Luma(tp), GW[i]);
        float  wn = BilateralW(lc, Luma(tn), GW[i]);
        diffused += tp * wp + tn * wn;
        w        += wp + wn;
    }
    diffused /= w;

    // ...luma gate math...

    float3 result = lerp(base.rgb, diffused, adaptive_str * diff_luma * luma_b);
    return float4(saturate(result), base.a);
```
`diffused` here is the full 2D separable bilateral blur (radius ≈ 51px at 2560w). The quantity `base.rgb − diffused` is the high-frequency Laplacian layer — computed implicitly but never stored.

### Proposed delta
After the `result` line (~line 196), add the bell-weighted clarity lift before returning:

```hlsl
    float3 result = lerp(base.rgb, diffused, adaptive_str * diff_luma * luma_b);

    // Clarity: Laplacian residual, bell-weighted to midtones (no gate)
    float3 detail = base.rgb - diffused;
    float  bell   = luma_b * (1.0 - luma_b) * 4.0;  // peaks at L=0.5, zero at 0 and 1
    result += CLARITY_STRENGTH * detail * bell;

    return float4(saturate(result), base.a);
```

Add to the Tuning section (~line 22):
```hlsl
#define CLARITY_STRENGTH  0.0    // 0–0.5; midtone local contrast lift (0 = bypass)
```

The `bell = luma * (1 − luma) * 4` function is the same shape used in `olofssonian_color_grade.fx:218` for the toe tint bell — a known-stable pattern in this codebase. It is self-limiting at both SDR boundaries: as `luma_b → 0` the bell → 0 (no darkening of black), as `luma_b → 1` the bell → 0 (no clipping of white). No conditional gate anywhere.

The pro-mist softening (`adaptive_str * diff_luma * luma_b`) operates in the highlight zone (luma 0.62–0.88). The clarity bell peaks at luma 0.50 and is near-zero above 0.90. These two effects occupy complementary luma regions with minimal overlap.

### Injection point
`general/pro-mist/pro_mist.fx`, lines 196–197 (DiffuseV pass, final result computation). Two lines inserted; one `#define` added to Tuning section. No new passes, no new textures.

### Breaking change risk
**Level:** Low
**Reason:** Default `CLARITY_STRENGTH 0.0` is full bypass — existing behavior unchanged. At non-zero values: (1) `detail` can be negative in pixels darker than the local bilateral average; `bell → 0` at luma extremes keeps these within SDR; (2) at `CLARITY_STRENGTH > 0.4` on highly saturated scenes, per-channel boost can cause hue shift — mitigated by keeping strength ≤ 0.3; (3) halos are suppressed by the bilateral weight in the blur (edge-preserving), so detail is small near hard edges.

### Viability verdict
**PASS** — operates in linear light, self-limiting at [0, 1] via bell, no gates, no ML, SPIR-V safe (no static arrays or `out` variable), zero additional GPU passes. The only new variable is `CLARITY_STRENGTH` with a safe default.

## Finding 2: CSF Dual-Scale DoG — Targeting the 3–5 cpd Perceptual Band

**Source 1:** https://dl.acm.org/doi/10.1145/3528223.3530115 — stelaCSF (ACM Transactions on Graphics, 2022)
**Source 2:** https://arxiv.org/abs/2502.13945 — GPU-Friendly Laplacian Texture Blending (Feb 2025)
**Year:** 2022 / 2025
**Field:** Perceptual vision science × real-time computational photography

### Core thesis
stelaCSF establishes that human contrast sensitivity has a characteristic bandpass profile peaking at approximately 3–5 cycles per degree (cpd) under photopic conditions. At a typical gaming display (27", 2560×1440, 60 cm viewing distance), 1 cpd ≈ 26 pixels; the CSF peak at ~4 cpd corresponds to a Gaussian blur radius of roughly 6–7 pixels. A Difference-of-Gaussians (DoG) using a fine bilateral blur (σ ≈ 6px) and a coarse bilateral blur (σ ≈ 26px) isolates exactly this perceptual clarity band — not pixel-level noise (finest details that look gritty) and not large-scale tonal movement (handled by `alpha_zone_contrast`). The GPU-Friendly Laplacian paper confirms the architecture: two separable bilateral passes at different radii, subtracted, is the correct way to extract a Laplacian band on GPU.

### Current code baseline
No spatial-frequency-selective contrast pass exists in the active alpha chain. `alpha_zone_contrast.fx` (lines 107–112) applies luma equalization globally through the full CDF:
```hlsl
    float equalized = tex2D(LumCDF, float2(luma, 0.5)).r;
    float new_luma  = lerp(luma, equalized, -(CURVE_STRENGTH / 100.0));
    float scale     = clamp(new_luma / luma, 0.0, 3.0);
    float3 result   = col.rgb * scale;
```
This moves all spatial frequencies together — a global luma scaling with no frequency discrimination. The 3–5 cpd perceptual peak is neither boosted nor protected.

### Proposed delta
New shader `general/clarity/clarity.fx`, inserted between `alpha_chroma` and `color_grade` in the effects chain.

```hlsl
#define CLARITY_STRENGTH   0.30   // 0–1; mid-frequency boost intensity
#define RADIUS_FINE        0.0025 // bilateral radius: fine   (~6px at 2560w, ~4 cpd at 27"/60cm)
#define RADIUS_COARSE      0.010  // bilateral radius: coarse (~26px at 2560w, ~1 cpd)
#define BILATERAL_K        25.0   // luma similarity falloff (same as pro_mist pattern)

// Gaussian weights, 5-tap sigma≈1.5 (same as pro_mist GW[])
static const float GW[5] = { 0.2270, 0.1945, 0.1216, 0.0540, 0.0162 };

// -- Pass 1: FineH  (bilateral H at RADIUS_FINE)  → FineTex
// -- Pass 2: FineV  (bilateral V at RADIUS_FINE)  → FineTex
// -- Pass 3: CoarseH (bilateral H at RADIUS_COARSE) → CoarseTex
// -- Pass 4: CoarseV (bilateral V at RADIUS_COARSE) → CoarseTex
// -- Pass 5: Apply
float4 ClarityPS(float4 pos : SV_Position, float2 uv : TEXCOORD0) : SV_Target
{
    float4 col    = tex2D(BackBuffer, uv);
    float3 fine   = tex2D(FineSamp,   uv).rgb;
    float3 coarse = tex2D(CoarseSamp, uv).rgb;

    // DoG: isolates the 3–5 cpd perceptual clarity band
    float3 band   = fine - coarse;

    float  luma   = Luma(col.rgb);
    float  bell   = luma * (1.0 - luma) * 4.0;   // self-limiting CSF luma envelope

    float3 result = col.rgb + CLARITY_STRENGTH * band * bell;
    return float4(saturate(result), col.a);
}
```

The `RADIUS_FINE = 0.0025` at 2560px width = ~6.4px, matching the 4 cpd spatial frequency of the stelaCSF peak. The `RADIUS_COARSE = 0.010` = ~25.6px ≈ 1 cpd. The DoG band `fine − coarse` is positive where there is mid-frequency structure and near-zero in flat regions — it applies zero clarity to flat grey, maximum clarity where contrast edges of the right spatial scale exist.

The bilateral weight in each blur pass: `gw * exp(−d² × BILATERAL_K)` with d = luma delta, identical to pro_mist's `BilateralW()`. This prevents halos at hard luminance edges.

Chain entry in `arc_raiders.conf`:
```
effects = primary_correction:frame_analysis:youvan:alpha_zone:alpha_chroma:clarity:color_grade:pro_mist:veil:output_transform
```

### Injection point
New directory `general/clarity/`, new file `clarity.fx`. One line added to `gamespecific/arc_raiders/arc_raiders.conf` effects= line. Technique is entirely self-contained — no shared textures with existing shaders.

### Breaking change risk
**Level:** Medium
**Reason:** Five new full-resolution passes at 1440p add ~4 ms at 60 fps target on mid-range GPU (each 9-tap separable bilateral = ~2M bilinear samples × 4 blur passes). Additionally: (1) `band = fine − coarse` goes slightly negative in zero-contrast regions — bell → 0 near luma = 0 suppresses this, `saturate()` catches any remaining undershoot; (2) hue shift is possible on highly chromatic edges at strength > 0.5 (a luma-preserving apply variant can fix this: scale by `new_luma / luma` rather than adding RGB directly); (3) cost is the main risk — Finding 1 should be validated first.

### Viability verdict
**PASS** — perceptually motivated, gate-free, self-limiting by construction, operates in linear light, no ML, SPIR-V safe. The GPU cost is real but bounded. Architecturally clean as a standalone shader.

## Strategic recommendation
Implement Finding 1 first: it is a two-line change to `pro_mist.fx` with zero GPU cost overhead, and directly answers whether the pipeline benefits from a positive clarity signal before committing to Finding 2's five-pass architecture. If `CLARITY_STRENGTH 0.15–0.25` visibly improves perceived detail in midtones without haloing (expected given the bilateral edge-preservation), then Finding 2 becomes the motivated upgrade — its scientific advantage is spectral precision, targeting the stelaCSF 3–5 cpd peak rather than boosting all frequencies above the bilateral cut. Implement Finding 2 only after Finding 1 is tuned and confirmed beneficial.
