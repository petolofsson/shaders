# Nightly Stability Audit — 2026-05-08

## Summary

Register pressure in `ColorTransformPS` is the primary crash suspect: ~168 scalar-equivalent
locals are declared, well above the 128-scalar driver-spill threshold; even with optimistic
block-scope reuse the estimate stays ~137–168 scalars. No outright NaN/INF paths were found —
all division, log, sqrt, and pow sites are guarded or rely on a guaranteed-positive domain.
BackBuffer row guards, temporal-history bounds, and the R19–R22 code are all correct.

## E. R19–R22 targeted review

### R21 — Hue rotation: zero-chroma case (C = 0)

When `lab.y = lab.z = 0` (achromatic pixel):

1. `ab_in = float2(0·r21_cos − 0·r21_sin, 0·r21_sin + 0·r21_cos) = float2(0,0)` — the
   sincos result is multiplied by zero; no NaN regardless of rotation angle.
2. `final_C = max(lifted_C, C)`. For C=0, `PivotedSCurve(0, m≥0, str)` = `saturate(0 +
   negative_term)` = 0 for all 6 bands, so `lifted_C = 0`, `final_C = 0`.
3. `C_safe = max(C, 1e-6) = 1e-6` (guarded).
4. `ab_s = float2(0,0) × (0 / 1e-6) = float2(0,0) × 0 = float2(0,0)` — not `0 × Inf`
   because `0 / 1e-6 = 0`, not `Inf`.
5. Output: `f_oka = f_okb = 0` — achromatic pixel preserved correctly.

**PASS** — zero-chroma case handled safely; no NaN via sincos path.

### R22 — Sat-by-luma: cannot produce negative C

```hlsl
C *= saturate(1.0 - 0.20 * saturate(1.0 - lab.x / 0.25)
                  - 0.25 * saturate((lab.x - 0.75) / 0.25));
```

- Inner shadow term ∈ [0, 0.20]; inner highlight term ∈ [0, 0.25].
- The two regions do not overlap: shadow is active for `lab.x < 0.25`, highlight for
  `lab.x > 0.75`. Maximum simultaneous sum = 0 (non-overlapping) → inner bracket minimum
  before outer `saturate` = 1 − 0.20 − 0 = 0.80 (or 1 − 0 − 0.25 = 0.75). Bracket
  minimum ≥ 0.75; outer `saturate` is conservative.
- C entering R22 = `length(lab.yz)` after Purkinje shift ≥ 0.
- C after R22 = C × (value ∈ [0.75, 1.0]) ≥ 0.

**PASS** — C cannot go negative.

### R19 — 3-way corrector: values stay in [0, 1] before Stage 2

`lin` is saturated by R50 (`lin = saturate(lin − …)`) before R19 runs, so `r19_luma ∈ [0, 1]`.

- `r19_sh = saturate(1 − luma/0.35)` ∈ [0, 1] ✓  
- `r19_hl = saturate((luma − 0.65)/0.35)` ∈ [0, 1] ✓  
- The two regions are non-overlapping (sh active below 0.35, hl above 0.65) → their sum ≤ 1
  for all `luma ∈ [0, 1]` → `r19_mid = 1 − r19_sh − r19_hl ∈ [0, 1]` ✓  
- Maximum total delta magnitude (TEMP=TINT=±100, extreme): R·(0.045, −0.030, −0.015).
  `lin = saturate(lin + delta)` clamps to [0, 1] before Stage 2. ✓  

**PASS** — R19 cannot push linear values below 0 or above 1 into Stage 2.

## Priority fixes

1. **CRITICAL — grade.fx `ColorTransformPS`: register spill**  
   Estimated ~168 scalar locals; optimistic reuse ~137. Both exceed the 128-scalar threshold
   where Vulkan drivers begin spilling to memory, causing severe slowdowns and intermittent
   GPU crashes — consistent with the post-R19–R22 symptom onset given that R21–R22 added
   several new locals to an already large function.  
   **Fix (low-risk, immediate):** Fold the 9 single-use intermediates listed in Section A
   (lum_att, spread_scale, k2, fla, one_mk4, cos_dt, sin_dt, lifted_C, C_safe) to reduce
   the upper bound to ~159 / optimistic ~128.  
   **Fix (definitive):** Split `ColorTransformPS` into two passes at the Stage 2→3 boundary
   (after `lin = lerp(lin_pre_tonal, lin, TONAL_STRENGTH / 100.0)`, before
   `RGBtoOklab(lin)`). Stage 3 passes `lin` through an intermediate texture.

2. **BENIGN — corrective.fx `UpdateHistoryPS` line 361: unguarded `RGBtoOklab` call**  
   `float3 rgb = tex2Dlod(BackBuffer, …).rgb;` followed immediately by `RGBtoOklab(rgb)` which
   calls `pow(l, 1/3)` without an explicit `max(0)` guard. Safe in practice (8-bit UNORM
   guarantees ∈ [0,1]) but not defensively coded.  
   **Fix:** Add `rgb = max(rgb, 0.0);` at grade.fx line 361 (before the `RGBtoOklab` call in
   `UpdateHistoryPS`).
