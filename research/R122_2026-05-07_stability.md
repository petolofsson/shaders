### R114 — Halation chromatic fringe ✗ TWO ISSUES

**Issue 1 — Green outer gain mismatch (grade.fx:317)**

```hlsl
// Actual code:
lin = saturate(lin + float3(hal_r, hal_g, hal_b) * float3(1.05, 0.45, 0.03) * HAL_STRENGTH);
// HANDOFF / CHANGELOG document:
// float3(1.05, 0.30, 0.03)
```

Green outer gain is **0.45**, not 0.30. This sits between the original 0.50 (pre-R114) and the
documented target of 0.30. R114 either reduced green partially and stopped, or the documentation
was set prospectively and the code was never updated. With `hal_g = hal_ring.g * lerp(0.78, 0.94, hal_lore)`,
the effective green weight ranges 0.35–0.42 — still warmer than the 0.23–0.28 range the docs imply.

**Issue 2 — hal_b Lorentzian lerp order reversed vs documentation**

```hlsl
// Actual code:
float hal_b = hal_ring.b * lerp(0.22, 0.38, hal_lore);
// HANDOFF documents:
// hal_ring.b * lerp(0.38, 0.22, hal_lore)
```

`hal_lore` is the Lorentzian tail weight (low = dark pixels near bright source, high = brighter
areas). Docs say blue fringe is stronger in the darker annular region (0.38) and weaker toward
bright cores (0.22). Code is reversed: blue is stronger toward bright cores (0.38) and weaker in
the dark annulus (0.22). The physical model is ambiguous here (blue scatter depth vs. yellow
filter attenuation), but the code and docs disagree — one is wrong.
