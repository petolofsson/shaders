# R37 — Film Halation
**Date:** 2026-04-30
**Type:** Proposal
**ROI:** High — biggest remaining perceptual delta. Immediately reads as "film" to the
eye. Arc Raiders has many point lights, neon signs, and bright windows that become halo
sources. Zero new textures, zero new passes.

---

## What halation is

When film is exposed, light that passes through the emulsion reflects off the film base
(or a backing layer) and re-exposes the emulsion from behind — most strongly in the
red channel because red dye sits in the deepest emulsion layer. The result: a soft warm
glow bleeding outward from bright highlights. It is not a lens flare; it is a smear that
lives in the same focal plane as the image.

Human perception: halation softens the harshness of clipped highlights and provides a
warm-light cue that reads as "photographic" rather than "rendered". It is one of the
most recognisable signatures of a film-like image.

---

## Signal

`CreativeLowFreqTex` (1/8-res, written by corrective.fx) is the scatter source. It is
already sampled in `pro_mist.fx` at mip 0. Halation needs a wider radius — mip 1 (1/16)
and mip 2 (1/32 effective) are available.

`PercTex.b` = p75 — the scene-adaptive highlight threshold.

---

## Physics → implementation mapping

On film, the red layer is deepest → scatters widest. Blue is shallowest → sharpest.

```
Channel  Emulsion depth  Scatter radius → mip level
R        deepest         widest          mip 2
G        middle          medium          mip 1
B        shallowest      narrowest       mip 0  (but attenuated)
```

This produces a warm (red-dominant) glow by construction — not by a colour tint but
by differential blur radii.

---

## Formula

```hlsl
// Sample three radii — one per channel, chromatic
float3 halo_r = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 2)).rgb;  // widest
float3 halo_g = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 1)).rgb;  // medium
float3 halo_b = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 0)).rgb;  // sharpest (already in ProMist)

// Take per-channel from the matching radius
float3 halo = float3(halo_r.r, halo_g.g, halo_b.b);

// Scatter delta — only add light, never subtract
float3 scatter = max(0.0, halo - base.rgb);

// Highlight gate — only pixels ABOVE p75 source the halo
float luma_in    = Luma(base.rgb);
float p75        = perc.b;
float halo_gate  = smoothstep(p75, p75 + 0.15, luma_in);

// Additive composite
float  HALATION_STR = 0.18;     // single new knob in creative_values.fx
float3 result = base.rgb + scatter * float3(1.20, 0.60, 0.25) * HALATION_STR * halo_gate;
```

The `float3(1.20, 0.60, 0.25)` channel weights impose the warm tint and reduce any
blue contribution. Combined with the chromatic radius map this gives a warm orange-red
bloom on highlights.

---

## Where to implement

**Option A — add to `pro_mist.fx`** (preferred):
`CreativeLowFreqSamp` and `PercSamp` are already declared. The halation block slots in
before the final saturate, as a second additive composite after the scatter/clarity block.
Single new knob: `HALATION_STR` (suggested default 0.18, exposed in `creative_values.fx`).

**Option B — new `halation.fx` effect:**
Cleaner isolation, own technique entry in the conf. Adds one technique pass (trivial
GPU cost). Preferred only if pro_mist becomes too large to read.

Option A is the right call here — pro_mist is 107 lines, well within readability bounds.

---

## Strength / scene adaptation

`HALATION_STR = 0.18` gives a subtle-to-moderate effect. It can be automated later
(R38) using `perc.b - perc.r` (scene dynamic range) — wider range → more halation.
For now a fixed knob is correct: it is a deliberate aesthetic dial, not a correctness
parameter.

---

## Interaction with pro_mist scatter

Pro-mist adds diffused light to midtones (gated around p75, looking downward in luma).
Halation adds diffused light to highlights only (gated above p75, looking upward).
They are complementary and additive — no coupling. The pro_mist `adapt_str` does not
need to change.

---

## Risk

**Clipping:** halation adds light additively. In SDR, `saturate()` on the final result
is the ceiling — no overflow. The `max(0, halo - base)` scatter delta is already
non-negative, so the worst case is reaching 1.0 and clamping. On an SDR pipeline this
is the correct behaviour (bright highlights bloom to white, which is physical).

**Gate seam:** `smoothstep` gate is seam-free by construction. No hard conditional on
pixel luminance.

## Success criteria

- Halation block added to `pro_mist.fx` after the clarity line
- `HALATION_STR` (default 0.18) in `creative_values.fx`
- Bright point lights / windows show a warm orange-red bleed outward
- Midtone and shadow content unaffected (gate = 0 below p75)
- No new textures, no new passes, no new technique entries
- Knob count: 20 + 1 = 21 (net neutral vs. current if R35/R36 already landed)
