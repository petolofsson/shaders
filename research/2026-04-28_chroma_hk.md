# Research Findings — Perceptual Chroma — 2026-04-28

## Search angle
Searched for psychophysical models that relate chroma/saturation to perceived brightness in
SDR displays, focusing on the Helmholtz-Kohlrausch (HK) effect — an effect our pipeline
currently ignores entirely despite already running per-band saturation curves. The cross-field
angle is from colorimetry and vision science (Nayatani 1997, Seong 2025) into real-time HLSL,
where the HK formula reduces to a handful of trig calls and one multiply.

---

## Finding 1: Nayatani (1997) Hue-Angle-Dependent HK Boost — Full Model
**Source:** https://github.com/ilia3101/HKE (implements Nayatani 1997, "Simple Estimation
Methods for the Helmholtz-Kohlrausch Effect", Color Research & Application 22:6)
Validated independently by: Bhaumik & Leloup 2025 (CIE 2025, P0132) and Seong & Kwak 2025
(doi:10.1002/col.22984).
**Year:** 1997 (model); 2025 (independent replication in VR HMDs and wide-gamut displays)
**Field:** Visual psychophysics / colorimetry, crossing into real-time rendering

### Core thesis
Saturated colors appear perceptually brighter than achromatic colors of identical luminance —
the HK effect. The magnitude depends on both chroma (CIELUV saturation s) and a hue-angle
factor q(θ) encoded as a Fourier series. At typical SDR viewing luminance (La ≈ 65 cd/m²),
a fully-saturated pixel appears ~20–24% brighter than its luminance value predicts. Our
saturation curves in `alpha_chroma_lift.fx` boost saturation without compensating for this
surplus perceived brightness, so increased saturation inadvertently inflates perceived luma —
an artefact the operator cannot currently see or correct.

### Current code baseline
File: `general/alpha-chroma-lift/alpha_chroma_lift.fx`, ApplyChromaLiftPS (~line 195–197)
```hlsl
    float final_sat  = (total_w > 0.001) ? new_sat / total_w : hsv.y;
    float3 processed = HSVtoRGB(float3(hsv.x, final_sat, hsv.z));
    return float4(lerp(col.rgb, processed, gate), col.a);
```

### Proposed delta
Insert between lines 196 and 197. K_Br is precomputed as a compile-time constant for
La = 65 cd/m²: `K_Br = 0.2717 * (6.469 + 6.362 * pow(65.0, 0.4495)) / (6.469 + pow(65.0, 0.4495))`
which evaluates to ≈ 1.098.

```hlsl
    float final_sat  = (total_w > 0.001) ? new_sat / total_w : hsv.y;
    float3 processed = HSVtoRGB(float3(hsv.x, final_sat, hsv.z));

    // Nayatani 1997 VCC: hue-angle-dependent HK compensation
    // θ is CIELUV hue angle; HSV hue * 2π is a close approximation in linear RGB.
    float theta  = hsv.x * 6.28318;
    float q_hue  = -0.01585
                   - 0.03017 * cos(theta)   - 0.04556 * cos(2.0 * theta)
                   - 0.02667 * cos(3.0 * theta) - 0.00295 * cos(4.0 * theta)
                   + 0.14592 * sin(theta)   + 0.05084 * sin(2.0 * theta)
                   - 0.01900 * sin(3.0 * theta) - 0.00764 * sin(4.0 * theta);
    float s_luv  = final_sat * 13.0;   // CIELUV saturation approximation
    float vcc    = 0.4462 * pow(1.0 + (-0.8660 * q_hue + 0.0872 * 1.098) * s_luv + 0.3086, 3.0);
    float hk_boost   = vcc / 0.999;   // normalized: = 1.0 for achromatic, ~1.20-1.25 at full sat
    float luma_scale = smoothstep(0.0, 0.35, Luma(processed));
    processed *= lerp(1.0, 1.0 / hk_boost, gate * luma_scale);

    return float4(lerp(col.rgb, processed, gate), col.a);
```

The `luma_scale` smoothstep prevents luma reduction in near-black chromatic pixels where
the HK effect is perceptually weaker. The division by `hk_boost` removes the surplus
perceived brightness that the saturation boost would otherwise add for free.

### Injection point
`general/alpha-chroma-lift/alpha_chroma_lift.fx`, lines 195–197 (ApplyChromaLiftPS).
Insert 8 lines between `float3 processed = HSVtoRGB(...)` and `return float4(...)`.

### Breaking change risk
**Level:** Low
**Reason:** At gate = 0 (near-grey or below SAT_THRESHOLD), processed is untouched. At full
gate and high saturation, luma is reduced by at most 1/1.25 ≈ 0.80 — a ~20% darkening of
highly saturated, mid-brightness pixels. This is perceptually neutral (the HK boost cancelled
out) but numerically visible in pixels that were previously benefiting from the free HK luma.
The hue-angle approximation (HSV → CIELUV angle) introduces ~5–15° angular error; main risk
is slight over/under-correction in the yellow-green band where q(θ) has its largest sin(θ)
peak (0.14592 coefficient).

### Viability verdict
**PASS** — Fully algebraic: 8 trig evaluations + arithmetic, no arrays, no branches on pixel
properties, no LUT. SPIR-V safe (all coefficients are inline literals). Output after
`* (1/hk_boost)` remains ≤ 1.0 since hk_boost ≥ 1.0. Linear-light safe. SDR by construction.

## Finding 2: Seong & Kwak (2025) — Saturation-Primary HK Model, Hue Independence
**Source:** https://onlinelibrary.wiley.com/doi/10.1002/col.22984
Seong, G. & Kwak, Y. (2025). "The CIECAM16-Based Lightness Model Incorporating the
Helmholtz-Kohlrausch (H-K) Effect Using Heterochromatic Brightness Matching Data."
*Color Research & Application.* doi:10.1002/col.22984
**Year:** 2025
**Field:** Display colorimetry / color appearance modeling

### Core thesis
Seong 2025 replicated the HK effect across four independent variables (saturation, test-color
luminance, background luminance, hue angle) and found that **hue angle has negligible influence
on H-K magnitude**, except for a small deviation near 180° (blue-green). The dominant driver
is saturation alone: J_HK = J + k · C, where J is CIECAM16 lightness and C is chroma.
This empirically eliminates the 8-term Fourier series of Nayatani (Finding 1) and replaces it
with a single linear coefficient — cheaper, more modern, and more conservative about hue
dependency.

### Current code baseline
File: `general/alpha-chroma-lift/alpha_chroma_lift.fx`, ApplyChromaLiftPS (~line 195–197)
```hlsl
    float final_sat  = (total_w > 0.001) ? new_sat / total_w : hsv.y;
    float3 processed = HSVtoRGB(float3(hsv.x, final_sat, hsv.z));
    return float4(lerp(col.rgb, processed, gate), col.a);
```

### Proposed delta
Two lines inserted between lines 196 and 197:
```hlsl
    float final_sat  = (total_w > 0.001) ? new_sat / total_w : hsv.y;
    float3 processed = HSVtoRGB(float3(hsv.x, final_sat, hsv.z));

    // Seong 2025: HK perceived-brightness surplus ≈ linear in saturation
    float hk_boost = 1.0 + 0.20 * final_sat;
    processed *= lerp(1.0, 1.0 / hk_boost, gate * smoothstep(0.0, 0.35, Luma(processed)));

    return float4(lerp(col.rgb, processed, gate), col.a);
```

The coefficient 0.20 corresponds to the approximate brightness-to-saturation slope observed
in heterochromatic brightness matching at La ≈ 65 cd/m² (derived from Nayatani's K_Br × 0.0872
× 13 ÷ (1 + 0.3086) ≈ 0.20). A `#define HK_STRENGTH 0.20` knob can expose this.

### Injection point
`general/alpha-chroma-lift/alpha_chroma_lift.fx`, lines 195–197, same window as Finding 1.
These two findings are mutually exclusive — implement one or the other, not both.

### Breaking change risk
**Level:** Low (lower than Finding 1)
**Reason:** The model is monotone in saturation with no hue-angle extremes, so there are no
bands that receive anomalously large or small corrections. Maximum luma reduction: 1/1.20 ≈
0.833 at final_sat = 1.0, gate = 1.0, bright pixel. Achromatic and near-grey pixels are
unaffected (gate ≈ 0).

### Viability verdict
**PASS** — Two lines. No trig. No arrays. No gates on pixel properties. SPIR-V safe. Bounded
output [0, 1]. Fully self-limiting: `gate` already handles the grey-floor exclusion.

## Strategic recommendation
**Implement Finding 2 first.** Two lines of code, zero trig, directly defensible by Seong &
Kwak 2025's empirical finding that saturation is the dominant HK variable. The coefficient
0.20 can be shipped as a `#define HK_STRENGTH` knob so the operator can dial from 0 (off) to
~0.30 (aggressive). Once that is validated perceptually, layer Finding 1's hue-selective
Fourier correction only if the blue-green band specifically shows residual overbrightness —
Seong 2025 predicts it will, at the 180° deviation they noted.
