## Candidate 1 — CHROMA_STR from HWY_ACHROM_FRAC (x=202)

**Verdict: FEASIBLE** — signal is valid, orthogonal to existing R151 adaptation, and available with no new infrastructure.

### Signal audit

`HWY_ACHROM_FRAC` (x=202) is written by `DebugOverlayPS` in `analysis_frame.fx:198`:

```hlsl
return float4(tex2Dlod(MeanChromaSamp, float4(0.5, 0.5, 0, 0)).a, 0.0, 0.0, 1.0);
```

The `.a` channel of `MeanChromaTex` is computed in `MeanChromaPS` (`analysis_frame.fx:340`):

```hlsl
achrom_frac = achrom_count / float(DS_W * DS_H);
```

where `achrom_count` accumulates pixels where `step(0.05, C) == 0`, i.e. Oklab C < 0.05, across the 32×18 downsample (576 samples). EMA-smoothed each frame (`analysis_frame.fx:346`):

```hlsl
float alpha = saturate(frametime * 0.005);   // ~0.083 at 60 fps → ~12-frame time constant
lerp(prev.a, achrom_frac, alpha)
```

No highway encoding needed — raw [0,1], no clip risk (`highway.fxh:29`).

### Orthogonality to existing R151 mean_C adaptation

Current `chroma_str_base` formula (`grade.fx:BuildSceneCtx`, lines 365–368):

```hlsl
float mean_C_scene  = ReadHWY(HWY_MEAN_CHROMA);
ctx.chroma_str_base = CHROMA_STR * 0.04
                    * lerp(0.80, 1.20, smoothstep(0.05, 0.35, ctx.zone_log_key))
                    * lerp(1.2, 1.0, saturate(mean_C_scene / 0.12));
```

`mean_C_scene` (HWY_MEAN_CHROMA, x=198) is the **median Oklab C of chromatic pixels only** (C≥0.05, `analysis_frame.fx:325`). `achrom_frac` is the **fraction of all pixels** below that same threshold. These are orthogonal measurements:

| Scene type | achrom_frac | mean_C_scene | R151 alone | proposed combined |
|---|---|---|---|---|
| Gray interior, vivid accents | high (~0.70) | moderate (~0.09) | 1.10× boost | 1.10 × 0.79 = 0.87× |
| Overcast outdoor (universal muted) | low (~0.25) | low (~0.06) | 1.15× boost | 1.15 × 0.93 = 1.07× |
| Vivid outdoor | low (~0.15) | high (~0.14) | 1.0× (floor) | 1.0 × 0.91 = 0.91× |

The target failure mode — gray interior where R151 over-boosts because `mean_C` of the few chromatic pixels is still moderate — is correctly attenuated by the achrom_frac term.

### Dark-scene conflation analysis

Near-black pixels have Oklab C typically < 0.05 regardless of hue, so a dark night scene can accumulate high `achrom_frac` through darkness rather than neutrality. However, three pipeline layers already attenuate the impact:

1. `zone_log_key` is low in dark scenes → the first lerp factor (`lerp(0.80, 1.20, ...)`) evaluates to ~0.80, already below nominal.
2. `LiftChroma` shape (`grade.fx:162–165`): `t = saturate(1 - C/pivot)`; vivid colored pixels (C near or above `pivot`) have t≈0 and are nearly unaffected regardless of `chroma_str`. The achrom_frac reduction acts mainly on low-C chromatic pixels, which genuinely need less lift in a dark scene.
3. The proposed floor is `lerp(1.0, 0.70, achrom_frac)` — at achrom_frac=0.6 (moderately dark scene) the multiplier is 0.82, which combined with the zone_log_key attenuation (~0.80) gives ≈0.66× nominal. Conservative enough to avoid crushing.

### Interaction with R151 — double-counting risk

In a gray-heavy scene, `mean_C_scene` of the chromatic pixels might be moderate (0.08–0.10), giving R151 a modest boost (~1.07–1.13×). The achrom_frac term partially reverses this. The combined effect is more accurate than either term alone: R151 sees only the colorfulness of the colored subset; achrom_frac sees the overall compositional "grayness." Neither alone captures both aspects.

Worst-case: a scene that is simultaneously high-achrom_frac (many neutrals) and low-mean_C (muted chromatics) → R151 gives 1.2× while achrom_frac gives ×0.70, net = 0.84×. This is correct — a very gray scene with weak color should get below-nominal chroma lift.

### Implementation sketch

In `grade.fx`, `BuildSceneCtx()`, after line 368:

```hlsl
// R173: compositional gray attenuation — fewer achromatic pixels needed for full lift.
// Orthogonal to R151 mean_C: R151 reads chromatic-pixel vibrancy; this reads overall
// scene composition. lerp floor 0.70 is conservative: max 30% reduction at achrom_frac=1.
ctx.chroma_str_base *= lerp(1.0, 0.70, ReadHWY(HWY_ACHROM_FRAC));
```

One `ReadHWY` call (single texel fetch, `highway.fxh:51`). No new textures, no new passes.

### Validation gap

The job file requires "validate correlation with subjective over-saturation before recommending." This cannot be resolved without frame capture. The formula is conservative (±30% at maximum) and the multiplier floor is 0.70 — a low-risk implementation-and-validate approach is appropriate rather than indefinite blocking. Recommend staging this alongside A/B capture on an overcast/interior scene vs. a night scene to confirm the dark-scene concern is not perceptible in practice.
