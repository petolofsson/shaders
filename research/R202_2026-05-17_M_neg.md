# R202 — Negative Spectral Sensitivity Matrix (M_neg)

**Date:** 2026-05-17  
**Status:** Proposal  
**Sources:** Kodak Vision3 500T datasheet (H-1-5219t), Gotanda SIGGRAPH 2010 "Film Simulation for Video Games", JanLohse/spectral_film_lut (GitHub), agx-emulsion/spektrafilm (Andrea Volpato, GPL v3)

---

## What it is

M_neg is the 3×3 input-side negative spectral sensitivity matrix. It models the fact that each dye layer in the film negative is not perfectly spectrally selective: the red layer has a small green sensitivity, the green layer picks up some blue, etc. These cross-layer exposures ride the H-D characteristic curve together with the primary exposure, producing film's characteristic "cross-hue richness" — reds that desaturate slightly toward orange, blues that soften toward cyan, without any of this being a simple saturation reduction.

The pipeline currently has the print-side matrix (R85 dye masking) but not the negative-side. Adding M_neg completes the physical two-stock chain: **negative sensitivity → H-D curve → print stock dyes**.

---

## Physical placement

In physical film:
1. Scene light × spectral sensitivity (M_neg) → per-layer exposure (linear)
2. H-D curve (FilmCurve) → per-layer density (log)
3. DIR couplers act during development (density space)
4. Print stock processing

**Pipeline insertion point:** after EXPOSURE and halation, immediately before `FilmCurveApply()` in `ApplyCorrective()`. Operating on `lin_e` in **linear sRGB** — not Oklab. Oklab is perceptually uniform and has no correspondence to physical film layer channels; mixing channels there would be physically meaningless.

```
lin_e *= E;                      // EXPOSURE
lin_e = ApplyHalation(...)       // optical scatter
lin_e = mul(M_neg, lin_e);       // ← INSERT HERE (linear sRGB)
fc_out = FilmCurveApply(lin_e);  // H-D curve
```

## Matrix values — Kodak Vision3 500T (5219)

Derived from Vision3 500T spectral sensitivity curves (H-1-5219t) digitized in spektrafilm v0.3.2 (Volpato 2026), using Gotanda's 3-point sampling method (sample each layer at 464 nm, 549 nm, 612 nm — the sRGB primary-adjacent wavelengths).

```
M_neg (rows = layer R,G,B ; columns = input channel R,G,B):

[ 1.0000   0.0227   0.0062 ]   ← red layer: 2.27% green pickup, 0.62% blue pickup
[ 0.0017   1.0000   0.0245 ]   ← green layer: 0.17% red pickup, 2.45% blue pickup
[ 0.0000   0.0002   1.0000 ]   ← blue layer: effectively pure
```

**Dominant terms:**
- Green-layer blue pickup: **2.45%** (largest cross-hue term)
- Red-layer green pickup: **2.27%**
- Red-layer blue pickup: **0.62%**
- Green-layer red pickup: **0.17%** (much smaller than community estimates of 1–2%)
- Blue layer: negligible cross-sensitivity by design (Kodak DLT technology)

**Note on estimates in the original request:** The "3–5% red-layer green" figure comes from spectral band integration over the full overlap region, not single-wavelength sampling. Band integration gives ~2.5× larger off-diagonal values. Gotanda's method (single wavelength at sRGB primaries) intentionally underweights the tails and is used here for consistency with his published method. The 3-point values are used as-is — they are conservative but directionally correct.

Vision3 250D (5207) has roughly half the off-diagonal values; 500T is the cinema standard and the more characteristic emulsion for this use case.

## HLSL implementation (no knobs)

```hlsl
// M_neg: Kodak Vision3 500T (5219) negative spectral sensitivity matrix.
// Rows = [R-layer, G-layer, B-layer], columns = [R-input, G-input, B-input].
// Derived from H-1-5219t spectral data (Gotanda SIGGRAPH 2010 sampling method).
// Applied in linear sRGB, after EXPOSURE + halation, before FilmCurve.
static const float3x3 M_NEG = float3x3(
    1.0000, 0.0227, 0.0062,
    0.0017, 1.0000, 0.0245,
    0.0000, 0.0002, 1.0000
);
lin_e = mul(M_NEG, lin_e);
```

`mul(M, v)` in HLSL: result[i] = dot(row_i(M), v) — correct for column-vector convention. Output is still in [R,G,B] channel order (layer R → display R, etc.).

**No knobs.** The matrix shape is a physical constant of the film emulsion — it does not drift across scenes and is not a creative parameter. The effect magnitude is baked into the datasheet values. The artistic control is the existing `CORRECTIVE_STRENGTH` blend weight that wraps the entire `ApplyCorrective()` output.

---

## SPIR-V compatibility

`static const float3x3` is on the CLAUDE.md silent-failure list — silently wrong output under SPIR-V. The matrix must be declared as a local `float3x3` inside the function body (not `static const`), or passed as a literal inline. Use a local declaration:

```hlsl
float3x3 m_neg = float3x3(
    1.0000, 0.0227, 0.0062,
    0.0017, 1.0000, 0.0245,
    0.0000, 0.0002, 1.0000
);
lin_e = mul(m_neg, lin_e);
```

## Expected visual effect

- **Saturated reds** → slight green-layer exposure bump → red shifts fractionally toward orange (correct: film reds are warmer than digital)
- **Blues** → green-layer picks up 2.45% → blues soften slightly toward cyan/teal (correct: film sky is less violently blue)
- **Greens** → minimally contaminated (green-layer red pickup is only 0.17%)
- **Overall:** gentle hue-specific desaturation and warmth — the "cross-hue richness" vs. a digital sensor's clean channel separation

Effect is small by construction (max off-diagonal 2.45%). It is not a dramatic color shift but a subtle texture change in how fully saturated colors read.

---

## Novelty

No game post-process pipeline implements the negative-side sensitivity matrix. R85 (dye masking) is the print stock side; this is the complementary negative side. Together they complete the physical two-stock chain. The method is published (Gotanda 2010) but has not been applied in a real-time shader.

Score impact: Stage 1 novelty +1–2% (currently 93%).
