# R56 — Film Halation

**Date:** 2026-05-01
**Status:** Proposal — needs findings before implementation.

---

## What film halation actually is

Halation is a scatter effect that occurs inside the film emulsion, not in the optics.
Light from a bright source exposes the emulsion front-to-back, reaches the film base,
and in older stocks reflects back from the base through the emulsion again. This
re-exposure hits the red dye layer hardest (it sits deepest, nearest the base) and
the blue dye layer least (it sits shallowest). The result is a tight warm/red fringe
tightly encircling the brightest highlights.

Key physical properties:
- **Tight radius** — film base is thin (~0.1 mm). Halation stays close to the source.
  This distinguishes it from bloom, which is wide and optics-driven.
- **Red-dominant** — red channel scatters furthest back through the emulsion; blue
  barely at all. A correct implementation has chromatic separation: wide-R, medium-G, no-B.
- **Threshold-gated** — only fires from very bright sources, not midtones or moderate
  highlights. Real film: only specular reflections and practical lights, not sunlit surfaces.
- **Additive only** — halation adds light; it never darkens.

## Current state (R37 inside pro_mist.fx)

Halation is currently baked into the pro_mist pass:

```hlsl
float3 halo_r    = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 2)).rgb;  // widest blur
float3 halo_g    = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 1)).rgb;  // medium blur
float  p75_gate  = max(perc.b, 0.55);
float  halo_gate = smoothstep(p75_gate, p75_gate + 0.15, luma_in);
float3 scatter_h = float3(halo_r.r, halo_g.g, diffused.b);
float3 delta_h   = max(0.0, scatter_h - base.rgb);
float auto_hal   = lerp(0.0, 0.22, smoothstep(0.55, 0.85, perc.b));
result          += delta_h * float3(hal_r, 0.60, hal_b) * auto_hal * halo_gate;
```

Problems with this implementation:
1. **Mip 2 is too wide for halation.** At 1/8-res base, mip 2 is 1/32-res — a very wide
   blur that overlaps with the game's own bloom radius. This is bloom stacking, not halation.
2. **Threshold too low.** `max(perc.b, 0.55)` fires in midtones in dim scenes. Halation
   should only activate above ~0.85 luma regardless of scene key.
3. **Blue channel gets diffused.b (mip 0)** — this gives blue a scatter it should not have.
   Correct: blue gets zero halation, or at most a tight mip 0 contribution scaled near zero.
4. **Coupled to pro_mist strength** (`auto_hal` is scene-key driven but not independently
   controllable). With pro_mist off, halation is also off.
5. **Not in active chain** — halation has never run independently of pro_mist.

## The "not ruining game bloom" problem

Arc Raiders has Lumen-driven global illumination and its own screen-space bloom. The game's
bloom is:
- Wide radius (several percent of screen width)
- Color-neutral to slightly warm
- Driven by the game's own HDR render before tonemapping

Our halation runs post-tonemapping on the SDR swapchain. If we use a wide scatter radius
(mip 2), we add a broad glow on top of the game's already-present bloom — this reads as
double-bloom. The solution is **tight radius only** (mip 0 or mip 1 at most), so halation
stays within the tight ring around the source where the game's bloom has not already
elevated the surrounding pixels.

Detection heuristic: `float3 already_lifted = max(0, diffused - base.rgb)` — where this
is large, the game bloom is present and halation should be attenuated. This is the same
delta used by the mist pass and is already computed.

## Constraints

- **Zero new passes, zero new textures.** `CreativeLowFreqTex` (mip 0–2) is already
  written by `corrective.fx` and already sampled in `grade.fx` (LOD 1 for Retinex).
  Halation can be added inside `ColorTransformPS` at zero additional GPU cost.
- **Must activate only on specular highlights** — luma threshold ≥ 0.85, hard floor
  regardless of scene key.
- **Must not stack with game bloom** — tight radius (≤ mip 1), attenuated where
  surrounding pixels are already elevated.
- **No gate (CLAUDE.md rule)** — use smoothstep thresholds, not hard conditionals on
  pixel values.
- **SDR ceiling** — all additions must be followed by `saturate()`.

---

## Proposed implementation location

Inside `grade.fx` `ColorTransformPS`, at the end of Stage 3 (CHROMA), after the
existing chroma/density work. `CreativeLowFreqTex` is already declared and sampled
at LOD 1 for Retinex. One additional `tex2Dlod` at LOD 0 for the tighter R channel.
Zero new infrastructure.

```hlsl
// R56: film halation — tight chromatic emulsion scatter from specular highlights only
float3 hal_src_r = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 1)).rgb;  // mip 1 — red spreads
float3 hal_src_g = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 0)).rgb;  // mip 0 — green tighter
float  hal_luma  = dot(col.rgb, float3(0.2126, 0.7152, 0.0722));
float  hal_gate  = smoothstep(0.85, 0.95, hal_luma);

// Attenuate where game bloom already lifted surroundings
float  bloom_present = saturate(dot(hal_src_g - col.rgb, float3(0.2126, 0.7152, 0.0722)));
float  hal_str   = (1.0 - smoothstep(0.0, 0.15, bloom_present)) * HAL_STRENGTH;

float3 hal_delta = float3(
    max(0.0, hal_src_r.r - col.r),  // red: widest, most scatter
    max(0.0, hal_src_g.g - col.g),  // green: tighter
    0.0                              // blue: none — film physics
);
col.rgb = saturate(col.rgb + hal_delta * float3(1.2, 0.5, 0.0) * hal_gate * hal_str);
```

## Research tasks

1. Find published emulsion cross-section diagrams (ARRI, Kodak) that confirm which dye
   layer sits deepest and quantify the relative scatter radii R:G:B.
2. Find cinema reference: at what display-referred luma does halation become perceptible
   on 35 mm prints? Is 0.85 the right threshold or should it be higher (0.88–0.92)?
3. Determine whether `HAL_STRENGTH` should be a fixed constant or a user knob. The
   current `auto_hal` (scene-key driven) is invisible to the user — research whether
   manual control adds value or just creates confusion.
4. Evaluate interaction with R55 pro_mist scatter: if both are active, do they compound
   at highlights? Propose a blending order that keeps them independent.
5. Write findings to `R56_2026-05-01_halation_findings.md`.
