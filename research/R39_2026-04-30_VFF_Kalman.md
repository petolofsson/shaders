# R39 — Variable Forgetting Factor Kalman (VFF)
**Date:** 2026-04-30
**Type:** Proposal
**Source:** R38 lateral research — Telecommunications (OFDM channel tracking)
**ROI:** High — scene-cut settling 20–40 frames → 2–5 frames, zero GPU cost, steady
scenes completely unaffected

---

## Problem

The Kalman filter in `SmoothZoneLevelsPS` and `UpdateHistoryPS` (R28) uses a fixed
process noise `KALMAN_Q = 0.0001`. This constant assumes the scene changes slowly
every frame. It is correct for smooth gameplay but wrong after a scene cut, where
zone medians can shift by 0.3+ luminance units in a single frame.

After a hard cut, the fixed-Q Kalman settles slowly — K_inf ≈ 0.095 means it
integrates ~10% of the new measurement each frame. Settling to within 5% of the true
value takes:

```
frames ≈ log(0.05) / log(1 - K_inf) ≈ log(0.05) / log(0.905) ≈ 30 frames
```

During those 30 frames, zone medians, Retinex blend, FilmCurve anchors, and chroma
history are all chasing stale data. Visible sluggish tone/contrast adaptation after
any hard cut.

The same problem affects `PercTex` (R34 Kalman in `CDFWalkPS`, Q=0.00005).

---

## Signal

The innovation (prediction residual) is already computed in the Kalman update:

```hlsl
float e = current.r - prev.r;   // already in the filter
```

A large `|e|` means the scene changed faster than Q predicted. This is exactly the
scene-cut detector — no new texture tap, no new pass, no new signal needed.

---

## VFF mapping from telecom

In OFDM channel estimation, the Variable Forgetting Factor tracks non-stationary
fading channels by modulating the forgetting factor λ from the prediction error:

```
λ(k) = λ_max − (λ_max − λ_min) · exp(−α · e(k)²)
```

Low `|e|` → λ → λ_max (slow adaptation, noise-resistant).
High `|e|` → λ → λ_min (fast re-lock, absorbs step change immediately).

In Kalman terms, λ maps to Q: lower λ → higher Q → larger K → faster adaptation.

For this pipeline the translation is direct — replace the fixed Q with a
residual-driven Q:

```hlsl
float Q_vff = lerp(KALMAN_Q_MIN, KALMAN_Q_MAX, smoothstep(0.0, VFF_E_SIGMA, abs(e)));
```

## Implementation

### `corrective.fx` — `SmoothZoneLevelsPS`

**Current:**
```hlsl
float P_prev = (prev.a < 0.001) ? 1.0 : prev.a;
float P_pred = P_prev + KALMAN_Q;
float K      = P_pred / (P_pred + KALMAN_R);
float median = prev.r + K * (current.r - prev.r);
float P_new  = (1.0 - K) * P_pred;
```

**Replacement:**
```hlsl
float P_prev = (prev.a < 0.001) ? 1.0 : prev.a;
float e      = current.r - prev.r;
float Q_vff  = lerp(KALMAN_Q_MIN, KALMAN_Q_MAX, smoothstep(0.0, VFF_E_SIGMA, abs(e)));
float P_pred = P_prev + Q_vff;
float K      = P_pred / (P_pred + KALMAN_R);
float median = prev.r + K * e;
float P_new  = (1.0 - K) * P_pred;
```

Retire `KALMAN_Q`. Add three defines:
```hlsl
#define KALMAN_Q_MIN    0.0001   // steady-state — identical to current
#define KALMAN_Q_MAX    0.10     // scene cut — K rises to ~0.91
#define VFF_E_SIGMA     0.08     // innovation scale: 0.08 luma unit triggers ramp
```

Apply the same pattern to `UpdateHistoryPS` chroma mean (replace its `KALMAN_Q` use).

### `analysis_frame.fx` — `CDFWalkPS`

Apply to p25/p50/p75 Kalman (R34). Use separate constants since percentile signals
move more slowly than zone medians:

```hlsl
#define KALMAN_Q_PERC_MIN  0.00005
#define KALMAN_Q_PERC_MAX  0.05
#define VFF_E_SIGMA_PERC   0.06
```

## Behaviour by scenario

| Scenario | `|e|` | `Q_vff` | K | Settling |
|----------|--------|---------|---|----------|
| Steady gameplay | < 0.01 | ≈ Q_MIN = 0.0001 | ≈ 0.095 | Same as now |
| Gradual exposure ramp | 0.03–0.06 | 0.001–0.02 | 0.15–0.65 | 3–8 frames |
| Hard scene cut | 0.15–0.40 | ≈ Q_MAX = 0.10 | 0.85–0.91 | 2–3 frames |
| Flash / specular spike | 0.40+ | Q_MAX | ~0.91 | 1–2 frames, then damps |

---

## Risk

**Specular spikes:** A single bright frame (explosion, muzzle flash) produces a large
`|e|` and briefly raises K. The filter absorbs the spike quickly then damps back as
`|e|` falls next frame. Net effect: the estimate transiently rises then falls — no
worse than the current EMA overshoot, and faster recovery. The CLAHE clip (R33)
limits the downstream S-curve slope, so the tonal consequence is bounded.

**P state after VFF step:** With Q_MAX = 0.10 and K ≈ 0.91, P_new = (1-K)*P_pred
≈ 0.009. Next frame, P_pred = 0.009 + Q_vff — if the scene settled, Q_vff drops
back to Q_MIN and K returns toward K_inf within 2–3 frames. No run-away state.

## Success criteria

- `KALMAN_Q` retired; replaced by `KALMAN_Q_MIN`, `KALMAN_Q_MAX`, `VFF_E_SIGMA`
- `SmoothZoneLevelsPS`, `UpdateHistoryPS`, `CDFWalkPS` all use residual-driven Q
- Steady-scene behaviour: indistinguishable from current (Q_vff ≈ Q_MIN when `|e|` small)
- Scene cuts: zone medians and PercTex anchors settle within 3–5 frames
- Zero new passes, zero new textures, zero new taps
- Net ALU delta: +3 ops per zone per frame (one subtract, one smoothstep, one lerp)
