## INVERSE_STRENGTH base adaptive derivation

### Current behaviour (`inverse_grade.fx:70–95`)

```hlsl
// inverse_grade.fx lines 76–85
float slope_enc = ReadHWY(HWY_SLOPE);    // highway x=197
float slope     = slope_enc * 1.5 + 1.0; // decode [1.0, 2.5]
...
float factor = lerp(1.0, slope, float(INVERSE_STRENGTH) * mid_weight * c_weight);
```

The slope is encoded in `analysis_frame.fx` DebugOverlayPS (highway x=197):

```hlsl
float log_iqr = log2(max(perc.b, 0.01)) - log2(max(perc.r, 0.01));
float slope   = clamp(2.5 / max(log_iqr, 0.5), 1.15, 1.8);
```

High IQR (wide tonal range) → slope near 1.15 (minimal expansion).
Low IQR (compressed scene) → slope near 1.80 (strong expansion).
IQR-based inverse scaling is fully encoded in `slope`. INVERSE_STRENGTH (0.50) is the
artistic scale: at INVERSE_STRENGTH=0.50 and slope=1.80, effective max expansion =
lerp(1.0, 1.80, 0.50) = 1.40×.

Additionally, the directional mean_C pivot (`inverse_grade.fx:92`) anchors expansion
relative to the scene chroma centroid — no per-scene calibration needed.

### Proposed formula

```hlsl
// Proposed (rejected — see below):
float log_iqr_live = ReadHWY(HWY_P75) - ReadHWY(HWY_P25);
float inv_need     = saturate(1.0 - log_iqr_live / 2.5);
float eff_inv_str  = INVERSE_STRENGTH * inv_need;
float factor       = lerp(1.0, slope, eff_inv_str * mid_weight * c_weight);
```

### Risk

Double-counting. Since `slope ≈ 2.5 / log_iqr`, multiplying by `1 - log_iqr/2.5 ≈ 1 - 1/slope`
makes effective expansion scale as `slope × (1 - 1/slope) = slope - 1`, which is a
super-quadratic response. At slope=1.80 (compressed), `inv_need ≈ 0.72`, effective factor
max = lerp(1.0, 1.80, 0.36) = 1.29× — weaker than the current 1.40×. At slope=1.15,
`inv_need ≈ 0.08`, factor max ≈ 1.006× — almost off. Counterproductive.

R102 verdict unchanged. No code changes since R102 affect this path.

### Verdict

**REJECT (confirmed from R102).** Slope already provides the first-order IQR-inverse
response. INVERSE_STRENGTH is an artistic scale on how much to trust the auto-derived
slope — not the same signal as scene compression.

## HAL_STRENGTH auto-enable

### Current behaviour (`grade.fx:516–542`)

HAL_STRENGTH is now **0.40** (re-enabled since the job spec was written; creative_values.fx
line 97 confirms). The auto-enable question is moot; the question becomes whether HAL_STRENGTH
should scale with scene highlight mass.

The halation block uses a dual-PSF model with a p90-adaptive threshold:

```hlsl
float hal_thresh = max(ReadHWY(HWY_P90) * 0.90, 0.50); // grade.fx:524
float hal_bright = smoothstep(hal_thresh, 1.0, hal_luma); // grade.fx:525
float hal_d      = 1.0 - hal_bright;
float hal_lore   = (HAL_GAMMA * HAL_GAMMA) / (HAL_GAMMA * HAL_GAMMA + hal_d * hal_d + 1e-6);
float3 hal_delta = float3(
    max(0.0, hal_ring_r + hal_wing_w.r * lerp(0.20, 0.42, hal_lore) - lin.r),
    max(0.0, hal_ring_g + hal_wing_w.g * lerp(0.10, 0.21, hal_lore) - lin.g),
    0.0
);
lin = saturate(lin + hal_delta * float3(hal_r_gain, hal_g_gain, 0.0) * HAL_STRENGTH);
```

### Proposed formula

```hlsl
// Proposed (rejected — see below):
float hal_auto = HAL_STRENGTH * smoothstep(0.35, 0.65, ReadHWY(HWY_P90));
lin = saturate(lin + hal_delta * float3(hal_r_gain, hal_g_gain, 0.0) * hal_auto);
```

### Verdict

**REJECT (confirmed from R102).** Three independent reasons:

1. `hal_delta = max(0, ...)` is zero everywhere the blurred image does not exceed the
   sharp pixel. In a dark scene, blur ≤ sharp at every pixel; delta evaluates to zero
   regardless of HAL_STRENGTH.
2. `hal_thresh = max(HWY_P90 * 0.90, 0.50)` already makes the brightness gate
   scene-adaptive: in a dark scene (p90=0.40), thresh=0.50 and hal_bright ≈ 0 for
   nearly all pixels. In a highlight-rich scene (p90=0.90), thresh=0.81, firing only
   on the scene's own top content.
3. Film sensitometry (Kodak/Fujifilm): halation magnitude is proportional to the local
   exposure integral above the antihalation threshold — per-pixel `max(0, blur−sharp)` is
   the correct physical model. No scene-level gate exists in any published sensitometric
   model; adding one would suppress halation in transitional-brightness scenes producing
   an unnatural step response.

## ZONE_STRENGTH inverse scaling

### Current behaviour (`grade.fx:257–258`)

```hlsl
float ss_08_25 = smoothstep(0.08, 0.25, zone_std); // grade.fx:253
float zone_str = lerp(0.26, 0.16, ss_08_25)
               * lerp(1.10, 0.93, lum_att) * ZONE_STRENGTH; // grade.fx:257–258
```

At zone_std = 0.08 (flat scene): base = 0.26. At zone_std = 0.25 (contrasty): base = 0.16.
That is 38.5% reduction from scene content alone, plus up to 15% further reduction from
`lum_att` in bright high-contrast scenes. Code unchanged since R102.

### Proposed formula

```hlsl
// Proposed (rejected — see below):
float zone_strength_eff = ZONE_STRENGTH / (1.0 + smoothstep(0.08, 0.25, zone_std) * 0.5);
float zone_str = lerp(0.26, 0.16, ss_08_25) * lerp(1.10, 0.93, lum_att) * zone_strength_eff;
```

At zone_std = 0.25: inner base = 0.16, outer divisor = 1.5 → effective = 0.16/1.5 × 1.15 ≈ 0.12.
Aggressively suppressed. Risks erasing legitimate tonal shaping in high-contrast exteriors.

### Verdict

**REJECT (confirmed from R102).** Inner formula already provides first-order inverse scaling.
ZONE_STRENGTH is the artist's outer multiplier for deliberate tonal control. Double-adaptation
would over-suppress at high contrast and remove the clean knob/auto separation.

### R101 F2 — H-K exponent scene-adaptation (NOT IMPLEMENTED — gap)

**Critical finding.** HANDOFF documents R101 F2 as shipped:

```hlsl
float hk_exp = lerp(0.52, 0.64, saturate(zone_log_key / 0.50));
float hk_boost = 1.0 + 0.25 * f_hk * pow(final_C, hk_exp);
```

Actual code `grade.fx:494`:
```hlsl
float hk_boost = 1.0 + 0.25 * f_hk * sqrt(final_C);
```

`sqrt(final_C)` = `pow(final_C, 0.5)`. Hellwig 2022 specifies exponent 0.587; R101 proposed
adapting this from 0.52 (dim scenes, stronger H-K) to 0.64 (bright exteriors, weaker H-K).
The current exponent 0.5 is below the dim-scene floor of 0.52, meaning H-K over-applies
in dim scenes relative to the R101 target. Error magnitude at C=0.15: ~4% above-target
H-K boost. Not visually catastrophic, but the Nayatani 1997 / CIECAM02 F_L backing cited
in R101 is not active.

**Ready formula (2 ALU, zero new textures):**

```hlsl
// Replace grade.fx line 494:
float hk_exp   = lerp(0.52, 0.64, saturate(zone_log_key / 0.50));
float hk_boost = 1.0 + 0.25 * f_hk * pow(max(final_C, 0.0), hk_exp);
```

`zone_log_key` is already in scope from the zstats read at `grade.fx:249`. `max(final_C, 0.0)`
guards the `pow` against negative input at zero C (safe; final_C = min(vib_C, C_ceil) is
non-negative by construction, but defensive is correct).

**Pumping risk:** `zone_log_key` is the Kalman-smoothed geometric mean of 16 zone medians
(~8-frame convergence). On a scene cut, K→1.0 overrides the Kalman in SmoothZoneLevels,
accelerating convergence. The hk_exp range 0.52→0.64 is narrow (12% swing), so the
transition produces ≤ 3% change in hk_boost at typical C values. Not perceptible on cuts.

## Literature findings

External search API unavailable (403 host_not_allowed). All findings from colour-science
corpus knowledge.

**Hunt effect and scene luminance (Hunt 1995, CAM16, CIECAM02):**
The Hunt effect is parameterised in CAMs via F_L (the adaptation luminance factor),
which scales the colourfulness response with scene luminance. R101 F2's adaptive H-K
exponent (0.52 for dim, 0.64 for bright) is directionally consistent with F_L: Nayatani
1997 and CIECAM02 both lower the chroma exponent in dim conditions to produce a stronger
relative boost. The unimplemented F2 gap is the most concretely literature-backed change
ready for implementation in the current pipeline.

**Inverse tone mapping and tonal statistics (Akyüz 2007, Banterle 2017):**
Published ITM methods consistently use IQR-derived spread as the primary adaptation
signal and apply it as a monotone slope — no second-layer gate. Confirms REJECT for
adaptive INVERSE_STRENGTH base.

**Halation physics (Kodak Eastman sensitometry, Fujifilm Technical Data):**
Antihalation dye absorption is quantified per-pixel relative to local exposure excess above
the absorption threshold. No scene-level gate appears in any published model. Per-pixel
`max(0, blur−sharp)` is the correct approximation. Confirms REJECT for HAL_STRENGTH
auto-enable.

**Zone contrast and scene adaptation (Adams 1980, Fairchild 2013 §15):**
Fairchild's scene-adaptive tone reproduction models treat a single scene-wide contrast
operator as the correct granularity for S-curve adaptation; per-zone pivots (already
implemented) are the finer level. A second outer scene-wide adaptation on ZONE_STRENGTH
is redundant. Confirms REJECT.
