# R23N — Pro-Mist Optical Model Research — 2026-04-29

## Summary

The current implementation has three scene-independent luma gate constants that place the scatter
activation window at the wrong luminance for any scene that isn't Arc Raiders at exactly the tuned
exposure. The kernel is a single-scale Gaussian when the correct optical model (confirmed by ProMist-5K,
arxiv 2601.19295) is a multi-scale sum of Gaussians — long tails are the defining characteristic of
Pro-Mist scatter. Both dedicated knobs (DIFFUSE_RADIUS, DIFFUSE_STRENGTH) can be eliminated; the
CLARITY_STRENGTH coupling to grade.fx is architecturally wrong and should be decoupled. The biggest
single visual upgrade is adding chromatic scatter weighting (R > G > B): this is what gives Black
Pro-Mist its warm highlight character and it costs zero GPU cycles.

---

## Literature

### ProMist-5K (arxiv 2601.19295, January 2025)
**"A Comprehensive Dataset for Digital Emulation of Cinematic Pro-Mist Filter Effects"**

The first principled physical model of Pro-Mist filter behaviour. Key findings:
- Uses **6 Gaussian blur layers at progressively larger kernel radii**, not a single Gaussian.
  Heavier filter density (#1/2) → larger-radius layers dominate. Lighter density (#1/8) →
  smaller-radius layers dominate.
- Operates in **scene-referred linear light** — no highlight gating. In linear light, bright
  pixels naturally dominate luminance-weighted scatter. Gating is redundant and introduces seams.
- Longer focal lengths → larger kernel radii (more pronounced scatter). Relevant if simulating
  specific glass; we will use scene statistics as the proxy instead.
- Hue remains relatively stable through the filter — the paper does not model chromatic scatter,
  which means that characteristic must come from halation physics (separate literature).

### Halation physics (Dehancer, prodigium-pictures.com)
Film layer structure directly explains chromatic scatter order:
- **Blue layer**: outermost (topmost emulsion layer) — filtered out first; least scatter
- **Green layer**: middle layer — receives scatter only when highlights are very bright
- **Red layer**: deepest layer (closest to camera back) — receives most bounce-back scatter;
  **dominant halation colour**

Black Pro-Mist warm highlight glow = R > G > B scatter weighting. This is confirmed by
cinematography.net: "the halation around a light caused by some older fog filters is bluer than
that caused by a ProMist filter." Tiffen's own page notes "the warm tone becoming more apparent"
in heavier densities. This is not a stylistic choice — it is the physical consequence of layer order.

### Froyok / UE4 Custom Bloom (froyok.fr, 2021)
Bloom post-process study. Relevant finding: **removing the luminance threshold entirely** and
blurring the full scene, then compositing, produces smoother and more stable results than
threshold-extracted bright passes. The multi-scale downsample/upsample pyramid (8 down, 7 up) is
the GPU-efficient implementation of the multi-Gaussian sum. The linear-in-linear-out composite
with weighted lerp between levels matches the ProMist-5K multi-scale architecture.

### Scatter kernel shape
The ProMist-5K 6-Gaussian architecture, the cinematography.net note that scatter is "slightly more
concentrated nearer the light source," and the general optics of polymer crystal diffusion all point
to a **long-tailed multi-Gaussian** profile rather than a single Gaussian. A true Lorentzian
(Cauchy) is not separable and has no finite variance, making it impractical for GPU use. The
sum-of-two-Gaussians (fine + coarse) is the correct real-time approximation: sharp core handles the
near-source concentration, wide tail handles the corona.

## Q1 — Scene-adaptive threshold (replacing DIFFUSE_LUMA_LO/HI/CAP)

### Current behaviour (grade.fx lines 138–139, pro_mist.fx)
```hlsl
float luma_gate = smoothstep(DIFFUSE_LUMA_LO, DIFFUSE_LUMA_HI, luma_in)
                * (1.0 - smoothstep(DIFFUSE_LUMA_CAP, 1.0, luma_in));
```
`DIFFUSE_LUMA_LO=0.55`, `DIFFUSE_LUMA_HI=0.65`, `DIFFUSE_LUMA_CAP=0.95` are scene-independent.
For a dark GZW night scene where p75≈0.35, the gate activates nowhere — zero effect. For a
bright outdoor scene where p75≈0.85, the gate activates at the midtone range, not highlights.

### Finding
ProMist-5K operates in linear light without any gate. The luminance-weighted scatter kernel
(`GW * (1 + Luma * LUM_BOOST)`) already ensures bright pixels dominate scatter naturally. The
gate is unnecessary and harmful. Eliminate it.

The near-clip fade (preventing blown whites from scattering into surroundings) is legitimate but
should be baked at (0.96, 1.0) since near-clip is resolution-independent. The scatter onset
should use `PercTex.b` (p75) as the scene anchor.

### Proposed formula (HLSL)
```hlsl
float4 perc       = tex2Dlod(PercSamp, float4(0.5, 0.5, 0, 0));
float  p75        = perc.b;
float  iqr        = perc.a;

// Onset tracks scene's upper quartile — activates in highlight range regardless of exposure
float  gate_lo    = saturate(p75 - 0.12);
float  gate_hi    = saturate(p75 + 0.06);
float  luma_gate  = smoothstep(gate_lo, gate_hi, luma_in)
                  * (1.0 - smoothstep(0.96, 1.0, luma_in));  // baked near-clip only
```
Three `#define` constants collapse to zero. Gate onset width (`0.12`, `0.06`) is fixed at
proportional 2:1 asymmetry — this matches the filter's physical behaviour (gradual onset,
sharper full-activation). Could be further simplified to no gate at all (ProMist-5K approach)
if temporal stability allows.

## Q2 — Kernel shape: multi-scale Gaussian using CreativeLowFreqTex

### Current behaviour (pro_mist.fx lines 83–104)
Single 9-tap Gaussian at radius `DIFFUSE_RADIUS / 4.0` in UV space. Provides only the fine
scatter component. No long tail.

### Finding
ProMist-5K uses 6 scales. For a 2-component GPU-efficient approximation we have a free coarse
component already in the pipeline: `CreativeLowFreqTex` is a 1/8-res version of the pre-grade
BackBuffer, computed every frame by corrective.fx. Sampling it at full UV gives a very wide
Gaussian equivalent (sigma ≈ 1/8 of frame width ≈ 240px at 1920w) at zero additional GPU cost.

The fine component (existing 9-tap full-res blur) handles the near-source concentration.
The coarse component (CreativeLowFreqTex) handles the corona tail.

### Proposed architecture
In `DiffuseVPS`, after computing `diffused` (existing fine blur), blend in the coarse component:

```hlsl
// Declare at top of file (texture already exists in corrective.fx — redeclare here)
texture2D CreativeLowFreqTex { Width = BUFFER_WIDTH/8; Height = BUFFER_HEIGHT/8;
                               Format = RGBA16F; MipLevels = 1; };
sampler2D CreativeLowFreqSamp { Texture = CreativeLowFreqTex;
                                 AddressU = CLAMP; AddressV = CLAMP;
                                 MinFilter = LINEAR; MagFilter = LINEAR; };

// In DiffuseVPS, after computing diffused:
float3 diffused_coarse = tex2Dlod(CreativeLowFreqSamp, float4(uv, 0, 0)).rgb;
float3 diffused_blend  = lerp(diffused, diffused_coarse, 0.30);
// 70% fine (near-source core) + 30% coarse (corona tail) → #1/2 density equivalent
```

**Blend ratio derivation**: ProMist-5K notes #1/2 density has larger-radius layers dominating
but not exclusively. 30% coarse is a conservative starting point. At 50% the visual character
shifts from subtle softening toward a more pronounced corona (approaches #1 density).

### Risk
CreativeLowFreqTex is written by corrective.fx before grade.fx runs. pro_mist.fx runs after
grade.fx. Need to verify the texture survives through the chain with correct content —
check that corrective.fx writes RGB (not luma-only) to the full 3 channels. If only luma is
stored in `.a`, the RGB channels may be zero, and this approach requires a corrective.fx change
to write full RGB. **Verify before implementing.**

## Q3 — Chromatic scatter weighting (warm highlight glow, zero knobs)

### Finding
Film layer physics (Dehancer, prodigium-pictures.com): red layer deepest → most scatter.
Blue layer outermost → least scatter. Black Pro-Mist confirmed warmer than other diffusion
filters (cinematography.net, Tiffen product page).

Physical calibration target: R receives ≈50–55% of scattered luminance, G ≈30–33%, B ≈12–17%.
This produces a warm shift in highlight halos without affecting neutrals (achromatic pixels
scatter equally across channels because `R==G==B`, so the weight vector is applied to the
scatter delta, not the base image).

### Proposed formula (HLSL)
Apply to the additive scatter composite (recommended) rather than the lerp composite:

```hlsl
// Scatter delta: only the light gain from diffusion (never darkens)
float3 scatter_delta = max(0.0, diffused_blend - base.rgb);

// Chromatic weights: R warm, B cool — baked from film layer physics
// Normalized so grey scatter sums to correct total energy
static const float3 CHROMA_SCATTER = float3(1.15, 1.00, 0.75);
// sum = 2.90/3 = 0.967 — very slightly subtractive overall (physical: scatter redirects, not adds)

float3 result = base.rgb + scatter_delta * CHROMA_SCATTER * adapt_str * luma_gate;
```

Note: using `max(0, diffused - base)` rather than `lerp(base, diffused, str)` is also
physically correct — a diffusion filter adds scattered light, it does not replace image content.
The current lerp can darken pixels at edges where `diffused < base`. The additive form eliminates
this artifact with no additional cost.

## Q4 — DIFFUSE_STRENGTH elimination

### Current behaviour
`DIFFUSE_STRENGTH * lerp(0.7, 1.3, saturate(iqr / 0.5))` — user sets the base, IQR scales ±30%.

### Finding
Black Pro-Mist #1/2 is the industry default (Blade Runner 2049, Dune, Oppenheimer). At #1/2
density on a calibrated monitor the scatter blend strength in linear light is approximately
0.15–0.20 of the scatter delta. Bake at 0.18 — the midpoint. IQR then provides the full
adaptive range: low-contrast scenes (flat game maps) get 0.126, high-contrast scenes get 0.234.
This covers #1/4 through #1 equivalent without a knob.

### Proposed formula (HLSL)
```hlsl
float iqr       = tex2Dlod(PercSamp, float4(0.5, 0.5, 0, 0)).a;
float adapt_str = 0.18 * lerp(0.7, 1.3, saturate(iqr / 0.5));
```
Remove `DIFFUSE_STRENGTH` from `creative_values.fx`.

## Q5 — DIFFUSE_RADIUS elimination

### Finding
ProMist-5K: longer focal length → larger kernel radii. In our pipeline we don't know focal
length, but scene brightness correlates — brighter highlights create more photon scatter energy
→ the effective corona radius grows. Use `p75 + iqr*0.5` as an approximation of p87 (upper
highlight quartile) to derive radius:

High-key bright scene (p75=0.80, iqr=0.30): p87≈0.95 → radius maps toward 0.020 (wide corona)
Typical Arc Raiders (p75=0.55, iqr=0.25): p87≈0.675 → radius ≈ 0.013 (moderate)
Dark GZW night map (p75=0.30, iqr=0.15): p87≈0.375 → radius maps toward 0.010 (tight)

### Proposed formula (HLSL)
```hlsl
float4 perc    = tex2Dlod(PercSamp, float4(0.5, 0.5, 0, 0));
float  p87     = saturate(perc.b + perc.a * 0.5);
float  r_uv    = lerp(0.010, 0.020, smoothstep(0.50, 0.90, p87)) / 4.0;
// Divided by 4 matches existing tap-offset convention
```
Remove `DIFFUSE_RADIUS` from `creative_values.fx`.

## Q6 — Clarity decoupling

### Current behaviour (pro_mist.fx lines 149–151)
```hlsl
float3 detail = base.rgb - diffused;
float  bell   = luma_in * (1.0 - luma_in) * 4.0;
result       += (CLARITY_STRENGTH / 100.0) * detail * bell;
```
Borrows `CLARITY_STRENGTH` from `creative_values.fx` — the same knob used by grade.fx Stage 2.
Changing grade clarity accidentally changes pro_mist clarity and vice versa.

### Finding
A diffusion filter physically *reduces* local contrast. The clarity recycling here is a practical
compensation (keeps detail after softening) not a physical phenomenon. It should be independent
of the grade's clarity knob. The correct strength for this compensation is proportional to how
much the diffusion blends in — more diffusion needs more compensation.

### Proposed formula (HLSL)
```hlsl
// Clarity proportional to scatter strength — auto-compensates for any strength level
float  clarity_w = adapt_str * 1.10;  // slightly over-compensate to keep perceived sharpness
float3 detail    = base.rgb - diffused_blend;  // use blended (fine+coarse) diffused
float  bell      = luma_in * (1.0 - luma_in) * 4.0;
result          += clarity_w * detail * bell;
```
Baked coefficient `1.10`: at `adapt_str=0.18` the clarity contribution is `0.198` — close to the
current effective value at `CLARITY_STRENGTH=35` (`0.35 * bell`), but now self-scales with
scatter strength. `CLARITY_STRENGTH` remains in grade.fx for the grade stage only.

| `DIFFUSE_STRENGTH` | User knob in creative_values.fx | **Removed** — baked 0.18 + IQR |
| `DIFFUSE_RADIUS` | User knob in creative_values.fx | **Removed** — derived from p87 |

## Implementation priority

1. **Q3 — Chromatic scatter** (highest ROI, zero cost): change `lerp(base, diffused, str)` to
   additive `base + max(0, diffused - base) * CHROMA_SCATTER * str`. This alone gives the
   warm highlight glow and eliminates the edge-darkening artifact. Two lines changed.

2. **Q4 + Q5 — Remove knobs**: bake DIFFUSE_STRENGTH and drive radius from PercTex. Confirm
   visual parity at Arc Raiders default values before removing from creative_values.fx.

3. **Q1 — Scene-adaptive gate**: replace the three `#define` constants with the p75-anchored
   smoothstep. Test on GZW dark scenes and Arc Raiders outdoor to verify correct placement.

4. **Q6 — Clarity decoupling**: small architectural fix. Low risk.

5. **Q2 — Multi-scale kernel** (biggest visual upgrade, requires verification): add
   CreativeLowFreqTex as the coarse component after confirming it carries full RGB in corrective.fx.
   If RGB is not available, this requires a corrective.fx change to write colour (not luma-only)
   to the low-freq texture — assess that separately.
