## Task B — Unsafe Math

Files scanned: grade.fx, corrective.fx, analysis_frame.fx, inverse_grade.fx, highway.fxh, hue_bands.fxh.

### Issues Found

**B1 — CONDITIONAL: `pow(max(lin, 0.0), EXPOSURE)` — grade.fx:373**  
`max(lin, 0.0)` guards against negative base (which would produce NaN for fractional exponents), but `pow(0.0, EXPOSURE)` = infinity when EXPOSURE < 0. If any pixel is exactly black (lin = 0) and the user sets EXPOSURE ≤ 0, output is inf. Risk is low (EXPOSURE negative is semantically nonsensical; likely UI-bounded in creative_values.fx which is not in the audit scope), but there is no in-shader domain check. Recommended: `pow(max(lin, 1e-6), EXPOSURE)` or a `clamp(EXPOSURE, 0.01, 4.0)` guard at the usage site.

### R126 — FilmCurve body S-curve

| Target | Status |
|--------|--------|
| `body_s = x*(1-x)*(1-2x)*0.12` | ❌ NOT present |
| Return: `x + body_s - factor*shoulder_w*above*above + toe_fac*toe_w*below*below` | ✓ grade.fx:158-159 |

**body_s deviation:** Job spec R126 specifies `x*(1-x)*(1-2x)*0.12`. Actual code (grade.fx:156-157):
```hlsl
float3 xw     = x * (1.0 - x);
float3 body_s = max(0.0, xw * xw * (2.0 * x - 1.0)) * 0.65;
```
This is `max(0, (x(1-x))²(2x-1)) * 0.65` — a strictly positive one-sided upper-mid lift, vs. R126's bidirectional `x(1-x)(1-2x)*0.12`. The CLAUDE.md documents the current formula explicitly ("upper-mid one-sided body lift `max(0,(x(1-x))²(2x-1))*0.65`"), confirming this is an intentional post-R126 revision. Not a regression.

**R126: MIXED** — return formula ✓; body_s formula is a post-R126 intentional revision consistent with CLAUDE.md documentation.
