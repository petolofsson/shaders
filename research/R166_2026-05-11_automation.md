### Signal complementarity

`HWY_MEAN_CHROMA` (x=198) measures mean Oklab C **of chromatic-only pixels** (C ≥ 0.05
threshold, `analysis_frame.fx:324`). It describes how saturated the coloured content is.

`HWY_ACHROM_FRAC` (x=202) measures the **prevalence** of near-neutral pixels regardless of
what the chromatic remainder looks like.

These are not redundant and can diverge significantly:

| Scene type | achrom_frac | mean_C_scene | Current net factor | With achrom factor |
|---|---|---|---|---|
| 80% gray walls + 20% normal-sat objects | ~0.80 | ~0.14 → lerp≈1.0 | 1.00 | 1.00 × 0.76 = **0.76** |
| 20% gray + 80% desaturated objects | ~0.20 | ~0.04 → lerp≈1.18 | 1.18 | 1.18 × 0.94 = **1.11** |
| 10% gray + 80% vibrant objects | ~0.10 | ~0.18 → lerp≈0.95 | 0.95 | 0.95 × 0.97 = **0.92** |
| Nearly all gray + trace muted color | ~0.95 | ~0.02 → lerp≈1.20 | 1.20 | 1.20 × 0.715 = **0.86** |

The first row is the primary target: a gray-heavy interior where R151's mean_C factor
leaves chroma lift unchanged (colorful objects have normal saturation), but the overall
scene context is predominantly neutral. achrom_frac correctly attenuates in this case.

The last row shows the worst-case compound: R151 boosts (muted color) while achrom_frac
cuts (nearly all gray). Net = 0.86×base, i.e., slightly below neutral. This is arguably
correct — a nearly achromatic scene does not benefit from aggressive chroma lift — but
the boost-then-cut interaction should be verified subjectively before shipping.

### Implementation sketch

`grade.fx:364–368`, add one line after the mean_C read:

```hlsl
float mean_C_scene     = ReadHWY(HWY_MEAN_CHROMA);
float achrom_frac      = ReadHWY(HWY_ACHROM_FRAC);     // new
ctx.chroma_str_base    = CHROMA_STR * 0.04
                       * lerp(0.80, 1.20, smoothstep(0.05, 0.35, ctx.zone_log_key))
                       * lerp(1.2, 1.0, saturate(mean_C_scene / 0.12))
                       * lerp(1.0, 0.70, achrom_frac);  // new: gray-heavy scenes get less lift
```

The highway debug write at `HWY_CHROMA_STR` (`grade.fx:655–658`) also needs updating —
it currently encodes only the zone_key factor (missing mean_C and would miss achrom_frac):

```hlsl
if (xi == HWY_CHROMA_STR) {
    float zk = tex2Dlod(ChromaHistory, ...).r;
    float mc  = ReadHWY(HWY_MEAN_CHROMA);
    float af  = ReadHWY(HWY_ACHROM_FRAC);
    float cb  = CHROMA_STR * 0.04

```

### Risks and mitigations

| Risk | Assessment |
|---|---|
| Signal fluctuation at scene cuts | EMA (α≈0.083/frame) absorbs brief transients; HWY_SCENE_CUT not wired to reset, same as mean_C |
| Compound attenuation with R151 | Worst case 0.84× base — modest; verify subjectively on gray-heavy interior vs. outdoor scene |
| CHROMA_STR is a locked artistic knob | Not violated — CHROMA_STR remains the user knob; formula only changes the scene-adaptive scalar |
| Highway slot 202 one-frame delay | Documented (`analysis_frame.fx:187`); same delay applies to HWY_MEAN_CHROMA — consistent |

**Recommendation:** Implement in a test branch. Compare gray-dominant interiors (high
achrom_frac) against outdoor scenes (low achrom_frac) for chroma over-lift regression.
The lerp floor of 0.70 is conservative; if testing shows the compound case is benign,
this can be shipped without further tuning.
