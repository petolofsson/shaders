# Nightly Stability Audit — 2026-05-09

## Summary

Register pressure in `ColorTransformPS` remains the dominant crash suspect and is
unchanged from R88: ~166–168 scalar-equivalent locals are declared, 38–40 above the
128-scalar driver-spill threshold that triggers register spilling on integrated and
mid-range Vulkan GPUs. No new unsafe-math paths were introduced; all BackBuffer row
guards, temporal-history bounds, and R19–R22 code remain correct. No source changes
were detected between R88 (2026-05-08) and this audit — findings fully confirm the
R88 baseline.

## E. R19–R22 targeted review

### R21 — Hue rotation: 2×2 rotation matrix and zero-chroma case

Matrix at grade.fx lines 371–376:
```hlsl
sincos(r21_delta * (0.10 * 6.28318), r21_sin, r21_cos);  // HLSL: sincos(x, &sin, &cos)
float2 ab_in = float2(lab.y * r21_cos - lab.z * r21_sin,
                      lab.y * r21_sin + lab.z * r21_cos);
```
- `sincos` argument order: first out-param receives sin, second receives cos → `r21_sin = sin(θ)`,
  `r21_cos = cos(θ)`. The rotation `(a·cosθ − b·sinθ, a·sinθ + b·cosθ)` is mathematically
  correct standard 2D rotation. ✓
- `sincos` is defined for all finite inputs — no domain restriction, no NaN regardless of
  rotation angle. ✓
- Zero-chroma case (C=0, lab.y=0, lab.z=0):
  - `ab_in = float2(0,0)` — sincos result multiplied by zero, result independent of angle ✓
  - `final_C = max(lifted_C, C) = 0` (PivotedSCurve(0,…) = 0 for all bands) ✓
  - `C_safe = max(0, 1e-6) = 1e-6` (division guard active) ✓
  - `ab_s = float2(0,0) × (0 / 1e-6) = float2(0,0)` — not 0×Inf because `0/1e-6=0` ✓
  - Output: f_oka = f_okb = 0, achromatic pixel preserved correctly ✓

**PASS** — zero-chroma case safe; no NaN via sincos path.

### R22 — Sat-by-luma: chained saturate cannot produce negative C

Expression at grade.fx lines 323–324:
```hlsl
C *= saturate(1.0 - 0.20 * saturate(1.0 - lab.x / 0.25)
                  - 0.25 * saturate((lab.x - 0.75) / 0.25));
```
- Shadow term (lab.x < 0.25): ∈ [0, 0.20]; highlight term (lab.x > 0.75): ∈ [0, 0.25].
- Regions are non-overlapping — never both non-zero simultaneously.
- Maximum deduction in any region: 0.20 (shadows) or 0.25 (highlights).
- Bracket minimum before outer `saturate`: 1 − 0.20 = 0.80, or 1 − 0.25 = 0.75. Both ≥ 0.
- Outer `saturate` is therefore conservative (redundant but harmless).
- C entering R22 = `length(lab.yz) ≥ 0`; C after R22 = C × [0.75, 1.0] ≥ 0.

**PASS** — C cannot go negative.

### R19 — 3-way corrector: linear values stay in [0, 1] before Stage 2

At grade.fx lines 272–283, `lin` is saturated by R50 (`lin = saturate(lin − …)`) before
R19 runs, so `r19_luma = Luma(lin) ∈ [0, 1]`.

- `r19_sh = saturate(1 − luma/0.35)` ∈ [0, 1] ✓
- `r19_hl = saturate((luma − 0.65)/0.35)` ∈ [0, 1] ✓
- Regions non-overlapping (shadow: luma < 0.35; highlight: luma > 0.65) → sum ≤ 1 for all
  luma ∈ [0, 1] → `r19_mid = 1 − r19_sh − r19_hl ∈ [0, 1]` ✓
- Maximum per-channel delta magnitude at TEMP=TINT=±100: ±0.045 (R), ±0.030 (G), ±0.015 (B).
- Final `lin = saturate(lin + Σdelta)` at line 282 clamps to [0, 1] before Stage 2 ✓

**PASS** — R19 cannot push linear values outside [0, 1] before Stage 2.

## Priority fixes

1. **CRITICAL — grade.fx `ColorTransformPS`: register spill (unchanged from R88)**
   ~166 scalar locals; optimistic reuse floor ~135. Both exceed the 128-scalar threshold.
   This is the most likely cause of the intermittent GPU crashes.

   **Immediate mitigation (low-risk):** Fold the 10 single-use intermediates listed in
   Section A (`lum_att`, `spread_scale`, `k2`, `fla`, `one_mk4`, `cos_dt`, `sin_dt`,
   `lifted_C`, `C_safe`, `log_R`). Upper bound drops to ~156, optimistic floor to ~125
   (below threshold).

   **Definitive fix:** Split `ColorTransformPS` into two passes at the Stage 2→3 boundary
   (after `lin = lerp(lin_pre_tonal, lin, TONAL_STRENGTH / 100.0)` on grade.fx:308, before
   `RGBtoOklab(lin)` on grade.fx:311). Stage 3 reads `lin` from an intermediate RGBA16F
   RenderTarget. This halves the live register count for each pass.

2. **BENIGN — corrective.fx `UpdateHistoryPS` line 360: unguarded RGBtoOklab input**
   `rgb = tex2Dlod(BackBuffer, …).rgb` fed directly into `RGBtoOklab` which calls
   `pow(l, 1/3)` without an explicit `max(0)` guard. Safe in practice (BackBuffer is
   8-bit UNORM [0,1]) but fragile. **Fix:** add `rgb = max(rgb, 0.0);` before the
   `RGBtoOklab(rgb)` call.
