## Task B — Unsafe math

**Status: ADVISORY — 2 low-risk unguarded sites; no critical issues**

**B-1 (low risk) — grade.fx:256 — unguarded sqrt in Apply3WayCC**
```hlsl
float r19_g = sqrt(dot(lin, float3(0.2126, 0.7152, 0.0722)));
```
No explicit `max(0, ...)` guard. Safe in context: `lin` is the output of `ApplyBleachBypass(out_lin, ...)` which returns `saturate(OklabToRGB(...))`, guaranteeing non-negative values. Dot product of non-negatives with positive weights cannot be negative. Zero input gives `sqrt(0) = 0`, which is safe for the downstream `saturate(r19_g / 0.35)` usage. **No NaN risk.**

**B-2 (low risk) — inverse_grade.fx:46 — atan2(lab.z, lab.y) without C > 0 guard**
```hlsl
float hue = frac(atan2(lab.z, lab.y) / 6.28318530);
```
When `C = length(lab.yz) = 0`, this calls `atan2(0, 0)`. In SPIR-V, `atan2(0, 0)` is implementation-defined but commonly returns 0 (not NaN). Downstream `HueSlopeBias(hue)` and `HueCeil(hue)` would operate on hue=0 (red band). The effect on output is benign because `c_weight = saturate((C - c_gate) / 0.15) = 0` when C < c_gate, making `factor = 1.0` and `new_C = C = 0`. Final output is protected by `saturate(OklabToRGB(lab))`. **No confirmed NaN path; output is safe.**

**No critical unsafe math sites found:**
- All `log2` calls are guarded: `log2(lin_e + 1e-5)`, `log2(max(..., 1e-10))`, `log2(max(..., 0.001))`, `log2(max(mode, 1e-6))`.
- All `pow` calls use `pow(max(..., 0.0), ...)` or `pow(max(..., 1e-10), ...)`.
- All division sites use `max()` epsilon guards.
- No `tex2Dlod(BackBuffer, ...)` detected in any audited file. (Zero-return vkBasalt bug: clean.)

### R126 — FilmCurve body S-curve (FilmCurveApply)

**Status: PARTIAL — return structure PASS; body_s formula DEVIATES from R126 spec**

- Return structure: `x + body_s - factor * shoulder_w * above * above + toe_fac * toe_w * below * below` (grade.fx:158-159) ✓

- **body_s DEVIATION:** R126 spec says `body_s = x * (1.0 - x) * (1.0 - 2.0 * x) * 0.12`. Actual code (grade.fx:156-157):
  ```hlsl
  float3 xw     = x * (1.0 - x);
  float3 body_s = max(0.0, xw * xw * (2.0 * x - 1.0)) * 0.65;
  ```
  This is `max(0, (x(1-x))²(2x-1)) × 0.65` — a one-sided squared version that lifts upper-mids only (zero at x ≤ 0.5, positive above). The R126 formula `x(1-x)(1-2x) × 0.12` is linear and bidirectional (a symmetric anti-S). These differ in shape, amplitude, and sign. CLAUDE.md documents the current formula as intentional: "upper-mid one-sided body lift `max(0,(x(1-x))²(2x-1))*0.65`". **Post-R126 change; job spec not updated.**
